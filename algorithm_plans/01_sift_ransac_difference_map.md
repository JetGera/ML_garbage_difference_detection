# План эксперимента: SIFT + RANSAC + difference map

## 1. Роль метода
Это самый интерпретируемый baseline для пар фото до и после уборки. Метод нужен как точка отсчета для проверки, насколько далеко можно продвинуться от простого выравнивания и пиксельного сравнения.

## 2. Когда метод уместен
1. Если сцена в двух снимках похожа по ракурсу.
2. Если есть достаточно текстурных деталей для сопоставления ключевых точек.
3. Если нужен объяснимый результат без обучения нейросети.

## 3. Данные
1. Пары фото до и после из локального `Dataset/`.
2. Небольшой контрольный набор внешних сцен для проверки устойчивости.
3. Для каждого изображения сохраняется исходная версия, выровненная версия и карта различий.

## 4. Окружение
1. Базовое conda-окружение из проекта.
2. OpenCV с поддержкой `SIFT_create`.
3. NumPy, SciPy, scikit-image, matplotlib, tqdm.
4. При необходимости использовать `opencv-contrib-python`, если SIFT недоступен в текущей сборке.

## 5. Pipeline
1. Привести обе фотографии к одному размеру и цветовому пространству.
2. Перевести изображения в grayscale.
3. При необходимости применить CLAHE или мягкое шумоподавление.
4. Найти SIFT keypoints и descriptors на обеих картинках.
5. Сопоставить дескрипторы через brute-force matching с L2 distance.
6. Применить ratio test, чтобы убрать слабые совпадения.
7. Оценить homography через RANSAC.
8. Выровнять второе изображение относительно первого.
9. Построить difference map через абсолютную разницу или структурную разницу.
10. Применить thresholding и морфологическую очистку.
11. Посчитать `change_ratio` или `area_ratio` в зависимости от выбранной логики.

## 6. Настройки, которые нужно подобрать
1. Минимальное число совпадений для запуска homography.
2. Порог ratio test.
3. Порог RANSAC reprojection error.
4. Размер ядра для морфологической очистки.
5. Способ пороговой бинаризации: Otsu или фиксированный порог.

## 7. Метрики
1. Количество инлайеров после RANSAC.
2. Средняя ошибка выравнивания.
3. Стабильность карты различий на похожих сценах.
4. Корреляция с ручной оценкой очистки.
5. Время работы на одном изображении.

## 8. Риски и ограничения
1. Метод ломается на бедных текстурой сценах.
2. Он чувствителен к сильному изменению освещения.
3. Большой параллакс или резкая смена ракурса ухудшают homography.
4. Мелкий мусор может теряться после выравнивания и пороговой обработки.

## 9. Что сохранить в отчете
1. Изображение матчинга ключевых точек.
2. Визуализацию homography и warped image.
3. Difference map до и после пороговой очистки.
4. Числа инлайеров, пороги и итоговые значения метрик.

## 10. Критерий успеха
Метод считается успешным как baseline, если он стабильно выделяет крупные изменения, дает понятную карту различий и служит честной отправной точкой для сравнения с нейросетевыми моделями.
