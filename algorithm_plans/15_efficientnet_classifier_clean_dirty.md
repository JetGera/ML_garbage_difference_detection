# План эксперимента: EfficientNet classifier для чисто/грязно

## 1. Роль метода
Это альтернативный бинарный классификатор для clean vs dirty. Он нужен, чтобы сравнить современную компактную архитектуру с ResNet и понять, дает ли compound scaling лучшее качество при меньшем размере модели.

## 2. Когда метод уместен
1. Если важен компактный и потенциально более эффективный классификатор.
2. Если данные ограничены, а модель не должна быть слишком тяжелой.
3. Если нужно сравнить качество и скорость с ResNet на одной и той же задаче.

## 3. Данные
1. Те же clean/dirty метки, что и для ResNet.
2. При необходимости тот же pseudo-label pipeline по `area_ratio`.
3. Внешние сцены можно использовать для расширения домена и проверки устойчивости.
4. Для TACO использовать `launcher/train_efficientnet_cls.py --taco-root ...`, который сам собирает clean/dirty crops из аннотаций.

## 4. Окружение
1. Легкое PyTorch-окружение.
2. torch, torchvision, timm, albumentations, numpy, opencv-python.
3. Отдельное логирование для сравнения с ResNet.

## 5. Подготовка данных
1. Использовать тот же split, что и у ResNet, чтобы сравнение было честным.
2. Подобрать аугментации без агрессивного разрушения сцены.
3. Сбалансировать классы, если dirty-кадров меньше.
4. Проверить, что модель не учит артефакты конкретной камеры или размера фото.

## 6. Pipeline обучения
1. Начать с `efficientnet_b0` как базового и компактного варианта.
2. При необходимости проверить `efficientnet_b3`.
3. Использовать ImageNet pretrained weights.
4. При маленьком датасете сначала заморозить backbone.
5. Применять cross-entropy или focal loss.

## 7. Pipeline инференса
1. Получить `dirty_prob` для каждого изображения.
2. Сравнить вероятность на фото до и после.
3. Использовать разницу вероятностей как один из вспомогательных признаков.
4. Сохранить результаты для сравнения с ResNet.

## 8. Метрики
1. Accuracy.
2. F1-score по dirty.
3. ROC-AUC.
4. Precision и Recall.
5. Скорость инференса и размер модели.

## 9. Риски и ограничения
1. Как и ResNet, модель дает только грубый глобальный сигнал.
2. Может быть лучше по параметрам, но не обязательно лучше по качеству.
3. При маленьком датасете преимущества EfficientNet могут быть незаметны.
4. Результат все равно должен использоваться только как дополнительный признак.

## 10. Что сохранить в отчете
1. Сравнение EfficientNet и ResNet по качеству и скорости.
2. ROC-кривые и F1 по dirty-классу.
3. Примеры предсказаний для типичных сцен.
4. Вывод о том, нужна ли эта модель в финальном pipeline.

## 11. Критерий успеха
Метод считается успешным, если он дает сопоставимое или лучшее качество по сравнению с ResNet при меньшем числе параметров или более высокой скорости.
