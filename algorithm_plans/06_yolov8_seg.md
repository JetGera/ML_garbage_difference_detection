# План эксперимента: YOLOv8-seg

## 1. Роль метода
Это быстрый instance segmentation baseline. Он нужен, чтобы получить маски мусора с минимальной сложностью запуска и сравнить их с Mask R-CNN и semantic segmentation.

## 2. Когда метод уместен
1. Если важен быстрый старт и простая эксплуатация.
2. Если нужны и bbox, и маски в одном решении.
3. Если хочется работать в одном семействе моделей YOLO для detection и segmentation.

## 3. Данные
1. TACO и другие наборы с polygon masks.
2. TrashCan как дополнительный источник.
3. Если локальный датасет без масок, использовать его только для тестовой проверки и визуальной оценки.

## 4. Окружение
1. YOLO-окружение.
2. ultralytics, torch, torchvision, pytorch-cuda.
3. opencv-python, pycocotools, albumentations.
4. tensorboard или wandb.

## 5. Подготовка данных
1. Привести аннотации к формату YOLO segmentation.
2. Проверить корректность полигонов.
3. Если классов много, начать с одного класса `trash`.
4. Разделить данные на train/val/test.

## 6. Pipeline обучения
1. Начать с `yolov8n-seg` или `yolov8s-seg`.
2. Подобрать размер входного изображения под мелкие объекты.
3. Использовать стандартные аугментации YOLO.
4. Следить за отдельно bbox и mask quality.
5. При необходимости сравнить несколько размеров модели.

## 7. Pipeline инференса
1. Получить masks и bbox для каждого изображения.
2. Преобразовать маски в площадь сегментации.
3. Вычислить `area_ratio` как изменение суммарной площади мусора.
4. При необходимости дополнительно считать `obj_ratio` по числу экземпляров.
5. Сохранить визуализации с контурами и overlay.

## 8. Метрики
1. Mask mAP.
2. Bbox mAP.
3. IoU и Dice по маскам.
4. Время инференса.
5. Ошибка по площади мусора.

## 9. Риски и ограничения
1. На маленьких объектах маски могут быть грубыми.
2. Шумная разметка ухудшает качество быстрее, чем в детекции.
3. Нужен хороший размер входа, иначе тонкие объекты теряются.
4. На сценах с сильным перекрытием результат может быть нестабилен.

## 10. Что сохранить в отчете
1. Примеры масок на типичных и сложных сценах.
2. Таблицу сравнения с Mask R-CNN и U-Net.
3. Показатели mask AP и `area_ratio`.
4. Логи обучения и параметры модели.

## 11. Критерий успеха
Метод считается успешным, если он дает достаточно точные маски при простой эксплуатации и становится практичным кандидатом в основной pipeline.
