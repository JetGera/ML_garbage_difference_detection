# План эксперимента: гибридный score

## 1. Роль метода
Это финальный слой принятия решения. Он объединяет результаты сегментации, change detection и detection в один числовой показатель, который можно использовать для оценки степени очистки и для начисления баллов.

## 2. Входы score
1. `area_ratio` из сегментации.
2. `change_ratio` из change detection.
3. `obj_ratio` из object detection.
4. При необходимости вспомогательные сигналы от clean/dirty classifier как quality check.

## 3. Логика построения
1. Каждый сигнал приводится к единому диапазону.
2. Сначала проверяется качество кадра и пригодность пары к автоматической оценке.
3. Затем рассчитывается комбинированный score.
4. После этого score переводится в понятные классы: clean, partially clean, dirty или approve, review, reject.

## 4. Подготовка данных для калибровки
1. Собрать на валидации предсказания всех моделей на одних и тех же парах.
2. Зафиксировать ручную оценку или эталонную метку очистки.
3. Подготовить таблицу, где для каждой пары есть все три сигнала и целевая оценка.

## 5. Первый вариант формулы
1. Начать с простой взвешенной суммы.
2. Вес каждого сигнала подобрать на валидации.
3. Сравнить линейную формулу с логистической регрессией или другим простым мета-моделем.
4. Не переходить сразу к сложной модели, пока не проверен простой вариант.

## 6. Этапы настройки
1. Нормализовать все признаки в один диапазон.
2. Проверить корреляцию признаков между собой.
3. Провести ablation, убирая по одному сигналу.
4. Настроить пороги для спорных случаев.
5. Зафиксировать финальную формулу и объяснение для диплома.

## 7. Метрики
1. Корреляция score с ручной оценкой.
2. ROC-AUC или accuracy для класса clean vs dirty.
3. Доля спорных случаев, отправляемых на ручную проверку.
4. Устойчивость к ошибкам отдельных моделей.
5. Калибровка и стабильность на разных сценах.

## 8. Риски и ограничения
1. Неправильные веса могут испортить даже хорошие базовые модели.
2. Сигналы могут конфликтовать друг с другом.
3. На маленькой валидации легко переобучить формулу.
4. Если входные модели слабые, score не сможет это компенсировать.

## 9. Что сохранить в отчете
1. Итоговую формулу score.
2. Таблицу весов и порогов.
3. Ablation results по каждому компоненту.
4. Примеры успешных и спорных решений.

## 10. Критерий успеха
Метод считается успешным, если итоговый score понятен, устойчив, хорошо объясняет качество очистки и реально улучшает принятие решения по сравнению с отдельными сигналами.
