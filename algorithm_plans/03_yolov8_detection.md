# План эксперимента: YOLOv8-detection

## 1. Роль метода
Это быстрый детектор мусора для получения числа объектов, их расположения и суммарной площади bbox. Метод нужен как основной detection-baseline для сравнения с двухэтапными детекторами и с instance segmentation.

## 2. Когда метод уместен
1. Если нужно быстро получить рабочий результат.
2. Если важны скорость, простота запуска и удобство воспроизведения на Windows.
3. Если мусор можно описывать ограничивающими рамками без обязательной масочной разметки.

## 3. Данные
1. TACO как главный внешний датасет.
2. TrashCan как дополнительный источник.
3. При наличии более простых датасетов их можно использовать для предварительного обучения.
4. Если локальный `Dataset/` не имеет bbox-разметки, использовать его для финальной проверки инференса и балльной логики, а не для первичного обучения.

## 4. Окружение
1. Отдельное YOLO-conda-окружение.
2. ultralytics, torch, torchvision, pytorch-cuda.
3. opencv-python, pycocotools, albumentations.
4. tensorboard или wandb для логирования.

## 5. Подготовка данных
1. Привести разметку к формату YOLO.
2. Если классов мусора много, сначала объединить их в один класс `trash`.
3. Сформировать train/val/test split.
4. Проверить баланс по размеру объектов и типам сцен.
5. Подготовить визуальный контроль нескольких размеченных изображений.

## 6. Pipeline обучения
1. Начать с pretrained `yolov8n` или `yolov8s`.
2. Заморозить часть слоев на коротком этапе, если датасет маленький.
3. Использовать стандартные аугментации YOLO: flip, scale, mosaic, color jitter.
4. Подобрать image size под мелкие объекты, при необходимости увеличить его.
5. Следить за early stopping и переобучением.

## 7. Pipeline инференса
1. Запустить модель на локальных парах фото.
2. Отфильтровать предсказания по confidence threshold.
3. Применить NMS.
4. Посчитать число объектов и суммарную площадь bbox.
5. Нормализовать показатель в `obj_ratio`.

## 8. Метрики
1. mAP50 и mAP50-95.
2. Precision и Recall.
3. FPS или среднее время инференса.
4. Ошибка по числу объектов и по суммарной площади.
5. Поведение на мелком и частично закрытом мусоре.

## 9. Риски и ограничения
1. Мелкие объекты плохо видны без увеличения разрешения.
2. Разметка bbox не дает точной площади мусора.
3. Сильная плотность мусора может давать перекрытия и нестабильный NMS.
4. Если локальные фото сильно отличаются от внешнего датасета, нужна аккуратная адаптация.

## 10. Что сохранить в отчете
1. Таблицу с метриками для разных размеров модели.
2. Примеры детекций до и после очистки.
3. Суммарные значения `obj_ratio` по тестовым парам.
4. Сравнение с Faster R-CNN и Mask R-CNN.

## 11. Критерий успеха
Метод считается успешным, если он дает стабильный и быстрый подсчет объектов, а его результат можно использовать в финальном гибридном score без сложной постобработки.
