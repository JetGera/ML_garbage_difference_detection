# План эксперимента: Faster R-CNN

## 1. Роль метода
Это двухэтапный детектор для сравнения с YOLOv8-detection. Он нужен как более традиционный и часто более точный baseline, особенно когда важнее аккуратность детекции, чем максимальная скорость.

## 2. Когда метод уместен
1. Если нужна более классическая детекторная архитектура.
2. Если хочется сравнить two-stage и one-stage подходы на одной разметке.
3. Если мусор бывает мелким, частично закрытым или сложным по форме.

## 3. Данные
1. TACO как основной источник bbox-разметки.
2. TrashCan как дополнительный набор.
3. При необходимости использовать часть локальных изображений только для проверки инференса и ручной оценки.

## 4. Окружение
1. Отдельное окружение Detectron2.
2. PyTorch и torchvision совместимой версии.
3. detectron2, fvcore, iopath, pycocotools.
4. OpenCV для визуализации и постобработки.

## 5. Подготовка данных
1. Привести разметку к COCO-формату.
2. Проверить категории и при необходимости объединить их в единый класс `trash`.
3. Сформировать train/val/test split.
4. Подготовить сводную таблицу размеров объектов и плотности сцен.

## 6. Pipeline обучения
1. Взять pretrained `Faster R-CNN R50-FPN` как стартовую точку.
2. При маленьком датасете сначала заморозить backbone, затем дообучить всю модель.
3. Подобрать learning rate, batch size и number of iterations под размеры GPU.
4. Отдельно проверить anchors и RPN proposal settings для мелких объектов.
5. Вести лог loss по RPN, classification head и box regression.

## 7. Pipeline инференса
1. Прогнать модель на тех же тестовых парах, что и YOLO.
2. Оставить только предсказания выше confidence threshold.
3. Применить NMS и получить итоговые bbox.
4. Посчитать число объектов и суммарную площадь боксов.
5. Преобразовать результат в `obj_ratio` для гибридного score.

## 8. Метрики
1. mAP50-95.
2. Precision и Recall.
3. Recall по мелким объектам.
4. Время инференса на кадр.
5. Ошибка оценки суммарной площади объектов.

## 9. Риски и ограничения
1. Модель тяжелее и медленнее YOLO.
2. Настройка Detectron2 на Windows может требовать аккуратной версии PyTorch и CUDA.
3. При ограниченной разметке модель может переобучаться на фон.
4. Для финального продукта may be overkill, но как эталонная точка сравнения полезна.

## 10. Что сохранить в отчете
1. Таблицу сравнения с YOLOv8-detection.
2. Примеры детекций на сложных сценах.
3. Логи обучения и значения основных метрик.
4. Итоговые `obj_ratio` по тестовым парам.

## 11. Критерий успеха
Метод считается успешным, если он дает более надежную детекцию на сложных изображениях и выступает сильным контрольным baseline для сравнения с YOLO.
